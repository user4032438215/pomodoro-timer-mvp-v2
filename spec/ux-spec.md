# UX 仕様書（ポモドーロタイマー）

## このアプリの至上命題
-   **ポモドーロテクニックに基づき、ユーザーの集中と達成感をサポートする**
-   ユーザーの集中を邪魔せず、作業負荷を増やさない
-   意味のある休憩を促し、終了時に達成感を得られるようにする

---

## MVP 機能（必須要件）
-   **タイマー開始**
    -  「25 分タイマー」ボタン
    -   作業時間、休憩時間、長い休憩時間、長い球形の頻度、それぞれをカスタマイズ可能にする。 ※未実装

-   **タイマー進行中の表示**
    -   残り時間をデジタルで表示（MM:SS形式） 
    -   現在のセッション状態（作業中／休憩中／長い休憩中）
    -   アナログ表記のUIも導入予定 ※未実装

-   **通知フロー**
-   タイマー終了時 → Web Notifications APIを使用して通知

-   **操作性**
    -   スタート／一時停止ボタン
    -   ポモドーロ終了ボタン（誤操作防止＋ナッジ付きダイアログ） ※未実装
    -   誤操作防止仕様：開始後は「停止／再開」「終了」以外の操作を非表示 ※未実装

-   **モチベーション演出**
    -   『現在〇ポモドーロ完了！』を画面上に表示

---

## 🐹 マスコット（マーモット）
- **終了通知** → 叫ぶマーモット  
- **休憩開始** → リラックスマーモット  
- **継続日数演出**（将来的に追加） → 旗を振るマーモット  

---

## 🔍 現在の検討ポイント
- セット数の履歴保存（ローカルストレージ or カレンダー連携）
- 「今日は〇分作業した」ログの蓄積と可視化
- ポモドーロ終了ボタンの配置と演出強化
- サインインオプション：Googleログイン＋カレンダー連携で作業時間を記録
- Web Notifications API通知に「次のタイマーへ」または「Later（あとで）」という選択肢を追加

---

## 🚀 将来的に追加したい機能（拡張要件）

### 1. 集中／休憩サポート
- タスク中：ブラウンノイズなど集中を促すBGM
- 休憩中：自然音などリラックスできるBGM

### 2. 強力な休憩促進機能
- 適切な頻度での音通知や追加演出

### 3. タスク管理機能
- Duolingo風「〇日継続中！」アニメーション
- GitHub草風のタスク量可視化

### 4. UIカラーマネジメント
- 作業中：低刺激・寒色系（例：H220, S15%, L40%）
- 休憩中：同じHで明度・彩度を変化させ、セッション変化を視覚的に伝える

### 5. その他拡張アイデア
- 詳細なタスク管理（期日・メモ入力）
- セッション時間／ポモドーロ回数の柔軟なカスタマイズ（実装予定）
